# Tech Skill - Patterns

## Design Patterns

## Pattern: YAGNI

**Trigger:** Adding features speculatively  
**Principle:** You aren't gonna need it  
**Action:** Only build what is required now

## Pattern: KISS

**Trigger:** Over-engineering  
**Principle:** Simple is better than complex  
**Action:** What is the simplest solution?

## Pattern: Separation of Concerns

**Trigger:** Mixed responsibilities  
**Principle:** Each module does one thing  
**Action:** Can this be split?

## Code Quality Patterns

## Pattern: Write Tests First

**Trigger:** Testing as afterthought  
**Principle:** Tests guide design  
**Action:** TDD for new features

## Pattern: Continuous Refactoring

**Trigger:** Technical debt accumulation  
**Principle:** Small improvements compound  
**Action:** Leave code cleaner than found

## Pattern: Code Reviews

**Trigger:** Solo coding  
**Principle:** Fresh eyes catch issues  
**Action:** Always get reviews

## Architecture Patterns

## Pattern: Make It Work, Make It Right, Make It Fast

**Principle:** In that order  
**Action:** Don't optimize prematurely

## Pattern: Prefer Explicit Over Implicit

**Principle:** Clear code beats clever code  
**Action:** Write for readers, not writers

## Pattern: Small Surfaces

**Principle:** Minimize public APIs  
**Action:** Encapsulate and hide implementation

## Career Patterns

## Pattern: Learn in Public

**Principle:** Teaching reinforces learning  
**Action:** Blog, share, contribute

## Pattern: Build Side Projects

**Principle:** Skills deepen through practice  
**Action:** Ship one project quarterly

## Pattern: Read Code, Not Just Tutorials

**Principle:** Real code teaches real patterns  
**Action:** Read popular open source projects
